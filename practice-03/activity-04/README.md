# Activity 04: Bluetooth, WiFi, and MQTT

This activity is split into three communication phases.

Expected subdirectories:

* `phase-a-bluetooth/`: Bluetooth connection between the ESP32 and a mobile device
* `phase-b-wifi/`: ESP32 WiFi web server for data visualization and output control
* `phase-c-mqtt/`: MQTT publisher/receiver workflow managed through a broker

Each phase should keep its own sketch, setup notes, and any local web assets required by the implementation.
