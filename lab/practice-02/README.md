# Electronic Instrumentation Laboratory

## Practice 2: Interrupts, Timers, Analog Inputs, and PWM Outputs in Embedded Systems

### 1. Objective

The purpose of this laboratory practice is to develop a solid understanding of the use of **interrupts**, **timers**, **analog inputs**, and **analog outputs through PWM** in **microcontroller-based embedded systems**, such as the **Arduino Uno** or equivalent platforms.

By the end of this practice, students should be able to:

* configure and use external interrupts,
* measure time intervals using interrupt-based techniques,
* acquire and process analog signals through the ADC,
* generate analog-like outputs using PWM,
* and present experimental results using both the **Serial Monitor** and the **Serial Plotter** available in the Arduino IDE.

For all activities described below, each group must use the **embedded platform that will be employed throughout the semester**.

### 3. Laboratory Activities

## Activity 1. External Interrupts

Students must study the concept of interrupts and their practical use in Arduino-based or equivalent embedded systems.

**Required task:**
Implement a pulse-counting system based on an interrupt-driven input. The pulses must be generated by a circuit that incorporates **debouncing**, whether by **hardware**, **software**, or a combination of both. The resulting pulse count must be displayed through the **Serial Monitor**.

**Expected outcome:**
A reliable pulse-counting implementation capable of rejecting false counts caused by switch bouncing.

---

## Activity 2. Interrupt-Based Timing

Students must study the use of interrupts for time measurement in embedded applications.

**Required task:**
Using the pushbutton and debouncing strategy implemented in Activity 1, develop a system in which each valid button event generates an interrupt and the **elapsed time between consecutive pulses** is measured. This information must be displayed through the **Serial Monitor**.

**Expected outcome:**
A system capable of accurately determining and reporting the time interval between valid input events.

---

## Activity 3. Analog Inputs and A/D Conversion

Students must study analog signal acquisition using the analog-to-digital conversion capabilities of the selected embedded platform.

**Required task:**
Implement the reading of **two LDRs (light-dependent resistors)** using the analog input channels of the embedded system. The measured values must be scaled appropriately in order to estimate the **resistance of each sensor** and determine the **relative position of the light source**. This information must be presented through the **Serial Monitor** and/or the **Serial Plotter**.

In addition, a **visual alarm system** must be implemented using one LED for each LDR, such that the LEDs indicate whether the light is predominantly concentrated on one side or the other.

**Expected outcome:**
A functional two-sensor light detection system capable of estimating light direction and providing both visual and serial feedback.

---

## Activity 4. PWM-Based Analog Output

Students must study the generation of analog-like output signals through **Pulse Width Modulation (PWM)**.

**Required task:**
Using the same embedded system and the analog information obtained in Activity 3, implement a PWM-based control signal to position a **90° or 180° servo motor** according to the detected direction of the light source. The system must also display, through the **Serial Monitor** and/or the **Serial Plotter**, the estimated **resistance of each LDR** and a corresponding **servo angle estimate** based on the detected light distribution.

**Expected outcome:**
A closed-loop light-following implementation in which the servo position responds to the comparative illumination detected by the two LDRs.

### 3.1 Repository Implementation for ESP32 + Arduino IDE

This folder now includes one self-contained Arduino sketch per activity:

* `activity-01/external-interrupts.ino`
* `activity-02/interrupt-timing.ino`
* `activity-03/dual-ldr.ino`
* `activity-04/light-following-servo.ino`

Each sketch is designed for an **ESP32 DevKit-style board** in the **Arduino IDE**.
The default pin assignment used by the implementation is:

* pushbutton input with pull-up: `GPIO 27`
* left LDR ADC input: `GPIO 34`
* right LDR ADC input: `GPIO 35`
* left indicator LED: `GPIO 18`
* right indicator LED: `GPIO 19`
* servo PWM output: `GPIO 23`

#### Wiring assumptions used by Activities 3 and 4

Each LDR is assumed to be used in a voltage divider with:

* LDR connected to `3.3 V`
* fixed resistor connected to `GND`
* ADC pin connected to the midpoint

The resistance estimate implemented in the sketches assumes a **10 kOhm fixed resistor**.
If your hardware uses a different resistor value or the divider is wired in the opposite order,
update the constants at the top of the corresponding sketch before running the lab.

#### Practical notes

* The pushbutton examples assume `INPUT_PULLUP`, so the button should connect the pin to `GND` when pressed.
* Activities 1 and 2 use **interrupt-level software debouncing** so they can be demonstrated without extra hardware.
* Activity 4 drives the servo using the ESP32 LEDC PWM peripheral and does **not** depend on an external Arduino servo library.
* Power the servo from an appropriate external supply when required, and always share **common ground** with the ESP32.
